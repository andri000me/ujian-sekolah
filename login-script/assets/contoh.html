DEVELOPMENT ENVIRONMENT

LINGKUNGAN PENGEMBANGAN FIREBASE

Untuk menyiapkan Lingkungan Pengembangan Firebase (Environment Development Firebase) sangat mudah hanya dalam beberapa langkah kita dapat menyiapkannya. Silahkan ikuti langkah-langkah dibawah ini.

Cara Menyiapkan Lingkungan Firebase, melalui tiga langkah sebagai berikut:

  1. Buat folder dengan nama public di dalam direktori Local Komputer yang dipilih.
  2. Buka dan jalankan Text Editor Favorite seperti Sublime text, VS studio, dan lainnya. Buatlah file index.html didalam text editor dan simpan di folder public. Silahkan Edit File HTML disamakan dengan mengikuti gambar dibawah ini:
  3. Install Node Package Manager (NPM) atau NODE JS yang berguna untuk menjalankan Firebase CLI. Sekarang jalankan terminal Command Prompt didalam direktori Local Komputer yang berisi folder public. Untuk lebih jelasnya lihat pada gambar dibawah ini

    4. Selamat tahap pertama sudah kita pelajari. Langkah selanjutnya, Kita akan belajar Bagaimana Menghubungkan Firebase CLI untuk dapat menggunakan layanan Hosting Firebase Gratis.

Firebase CLI

INTEGRASI WEB DENGAN FIREBASE CLI

Apa itu integrasi Firebase CLI ? Mari kita bahas singkat saja dan tidak membuang waktu, secara sederhana dapat diartikan sebagai jembatan yang menghubungkan 2 tempat. Yakni, antara firebase CLI dengan website kita. Sebelum masuk pada tahap integrasi pastikan kita memiliki akun firebase dan telah membuat project. Jika belum buat silahkan kunjungi https://console.firebase.google.com/.

Dengan adanya CLI di firebase kita dapat mengakses database dan hosting yang bersifat clouds, dikarenakan sudah terhubung dengan Command Line Interface (CLI). Sudah paham ? Let’s Go ! praktekan.

Mari lakukan langkah-langkah Berikut:

  1. Langkah pertama yang paling penting aktifkan terlebih dahulu aplikasi Web Project di akun Firebase kita. Lihat step by step gambar dibawah ini:

  2. Setelah mengaktifkan Web Aplikasi di Akun Firebase kita jalankan Terminal Command Prompt yang mengarah pada direktori folder bernama public dan berisi file index.html didalamnya.

  3. Dengan memberikan perintah $npm install –g firebase-tools, terminal kita akan mendownload secara otomatis file yang dibutukan untuk integrasi(Hosting Connection). Lakukan perintah Install firebase-tools dengan ketikan: $npm install –g firebase-tools.

  4. Tunggu sampai proses selesai, membutuhkan waktu 2-5 menit tergantung koneksi masin-masing.

  5. Setelah selesai, ketikan perintah: $firebase login untuk proses otentikasi data akun yang telah terdaftar. Secara otomatis akan di arahkan ke browser default untuk proses login menggunakan akun google.

  6. Selanjutnya, Jalankan perintah $firebase init untuk proses inisialisasi project firebase yang kita buat. Ikuti saja terlebih dahulu perintahnya sampai selesai dengan menekan tombol ENTER di keyboard. Kita akan membahas lebih dalam pada langkah berikutnya.

  7. Selamat tahap integrasi sudah berhasil selanjutnya kita akan melanjutkan Bagaimana Proses Inisialiasi Project Untuk Menggunakan Hosting Firebase.

INISIALISASI HOSTING

INISIALISASI FIREBASE HOSTING

Inisialisasi hosting firebase merupakan tahap lanjutan dari langkah-langkah sebelumnya telah kita lakukan dengan perintah $firebase init, Perintah tersebut merupakan bagian dari inisialisasi hosting. Inisialisasi Hosting Firebase merupakan cara memilih project hosting website kita, Inisialisasi ini akan dieksekusi oleh server Firebase CLI.

Pada tahap ini kita akan mengenal lebih dalam bagaimana inisialisai berjalan. Dengan demikian pada saatnya kita mengatur hosting website secara leluasa. Coba sekarang buka lagi folder public yang sebelumnya kita buat. Apakah ada perubahan atau penambahan file setelah melakukan perintah $npm firebase init ? tentu, sudah pasti kita melihat penambahan file seperti: firebase.json, .firebaserc, index.html dan 404.html.

File tersebut merupakan buatan firebase secara default atau otomatis. Sebetulnya tanpa membuat index.html diawal panduan, firebase akan membuatkannya secara otomatis. Namun kita ingin membuat index.html dengan costume sendiri. Ketika memberikan perintah $firebase init dan telah membuat file index.html,

Ada beberapa poin yang biasa ditanyakan ketika melakukan proses inisialisasi project perhatikan poin-poin dibawah ini:

  1. Memilih Nama Project Firebase Hosting

    Sebelumnya kita telah membuat nama project di akun firebase. Saat kita melakukan inisialisasi dengan menjalankan perintah $firebase init, Firebase CLI akan memberikan pilihan kepada user untuk menggunakan beberapa project yang telah dibuat dengan menekan tombol ENTER untuk memilih. Apabila kita membuat banyak project dan memutuskan memilih project yang lain gunakan tanda panah Atas/Bawah pada keyboard dan tekan ENTER.

  2. File HTML Default Yang Di Hosting

    File public/index.hml already exist. overwrite y/n ? (File index.html didalam folder public sudah ada, apakah kamu ingin menulis ulang index.html ? ya/tidak).

    Kita ketikan No karna sudah dibuat diawal. Kalau belum membuatnya ketikan Yes. Mudah bukan ?

    3. Untuk file 404.html hampir sama dengan proses inisialisasi index.html, yang membedakan nya disini kita akan diberikan pilihan untuk membuat Default Web atau Single Web. Jika memilih default web maka firebase akan membuatkan file 404.html. Sedangkan Single Web firebase tidak akan membuatkan file tersebut. Ingat tinggal ketikan yes atau no saja di dalam terminal.

    File Tambahan Wajib

    • Harus diingat ada 4 file tambahan wajib ada dalam project web hosting adalah firebase.json, .firebaserc, .gitignore dan firebase-debug. Tanpa file-file tersebut hosting tidak akan berjalan dengan baik. Tidak usah khawatir file tersebut akan dibuatkan secara default saat kita melakukan perintah $firebase init.

    Selamat kita telah belajar bagaimana proses inisialiasai bekerja dan memahami system default inisialisasi firebase. Selanjutnya kita akan belajar Mengaktifkan Mode Pengembangan firebase local.

MODE DEVELOPMENT

MENGAKTIFKAN PENGEMBANGAN LOKAL

Mengatur Mode Pengembangan Firebase dengan mengaktifkan server localhost untuk melakukan permintaan melalui api dan testing aplikasi web kita ketikan perintah $firebase serve didalam terminal aktif dengan direktori folder default.

Tunggu sampai proses selesai. Maka CLI akan memberikan alamat URL https://localhost:5000.

Silahkan buka di browser, dengan membuka URL tersebut. Coba edit file index.html kita dapat melihat perubahan ketika mengubah kode html di dalam text editor dan hasilnya akan dikirimkan ke alamat tersebut.

Bagiamana hasilnya sudah terlihat ? Jika belum berarti project belum terinisialisasi. Silahkan ulangi proses inisialisasinya. Apabila sudah terlihat, lanjutkan panduan berikutnya Bagaimana Cara Mengatur Banyak Multi Situs Firebase Hosting.

MULTI SITUS FIREBASE

MENGATUR HOSTING MULTI SITUS

Ok bagaimana sudah mempraktekannya dengan baik ya.. ? Saya kira begitu. kalau belum paham langkah-langkah sebelumnya, mungkin pada tahap ini akan lebih sulit. Sebaliknya kalau sudah paham semua langkah sebelumnya, tahap ini sangatlah mudah. Untuk mengatur hosting multi situs kita harus memahami isi firebase.json pada saat file json tersebut terinisialisasi.

Coba sekarang buka lagi file folder public didalam sublime text. Buka isi firebase.json, dan pahami kodenya. Secara umum firebase.json akan menulis default hosting berisikan kode seperti pada gambar dibawah ini :

Kita akan focus pada data json dengan nama public : public dimana kita menyimpan data asset file index.html. Untuk data yang lainnya abaikan saja dulu, bentuk data yang bertanda merah memberitahukan kepada firebase bahwa folder public yang ingin kita upload filenya.

Data json tersebut akan di eksekusi saat proses inisialisasi hosting. Jadi kesimpulannya kita dapat mengatur nama folder mana yang ingin di upload ke firebase hosting. Kemudian kita dapat memindahkan source website ke project yang lainya dengan membuatkan perintah firebase init lagi. Tetapi menggunakan perintah tersebut akan lebih lama, kita akan menggunakan cara cepat yakni mengganti konfigurasi nama Status Project Hosting didalam file tambahan firebaserc yang telah terinisialisasi. Sesuaikan Nama Status Project Default berdasarkan project masing-masing.

Nah bagaimana caranya kita membuat banyak situs di firebase ? Jawabannya gampang banget !. Silahkan masuk ke akun firebase, Masuk ke project sudah kita buat, Buka menu hosting tambahkan situs lainnya. Lihat dan Ikuti langkah-langkah dibawah ini sampai selesai.

Selamat tahap mengatur dan menaktifkan multi situs sudah kita kuasai kalau belum jangan ragu untuk tanya ya. Mari kita lanjutkan bagaimana web statis buatan kita dapat di akses secara umum oleh orang-orang di internet. Yakni, upload asset folder public ke firebase hosting. Ikuti Panduan selanjutnya. FIREBASE DEPLOY

DEPLOY WEB KE HOSTING FIREBASE

Pada langkah sebelumnya sudah kita bahas bagaimana membuat mode pengembangan atau testing web statis dengan server firebase. Pada tahap ini kita akan belajar bagaimana men-deploy web statis ke firebase hosting. Apa sih deploy itu ? deploy kebalikan dari istilah mode pengembangan. Sudah jelas ya.. Jadi deploy itu hasil production website (Web jadi). Website yang sudah kita buat dan yakin web tersebut ingin kita upload ke hosting serta siap dilihat oleh banyak orang. Pada tahap terakhir Firebase E-book Serie A ini saya akan memberikan contoh website Login. Ok ! Mari kita praktek.

Pastikan file asset seperti index.html, 404.html dan lainnya yang ada didalam folder sudah diedit dan siap untuk di upload (deploy). Masuk ke Terminal Command Prompt yang masih aktif pada project public folder, ketikan perintah: $firebase deploy. Tunggu sampai proses upload selesai. Setelah itu buka di browser URL hosting yang telah berhasil kita deploy.
